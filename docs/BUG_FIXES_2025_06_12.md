## 1. Batch Compare Resilience

**Issue**: If any single keyword lookup failed in `batchKeywordDifficulty`, the entire batch request failed, resulting in zero results and a confusing error to the user.

**Fix**: `packages/api/src/services/keyword-service.ts`
- Added try/catch in the batch loop to silently skip failed keywords
- Batch now returns partial results (successful lookups only)
- Caller sees what succeeded instead of complete failure

**Code**:
```typescript
for (const item of items.slice(0, 25)) {
  try {
    results.push(await getKeywordDifficulty(item.keyword, item.country, item.store));
  } catch {
    // Silently skip failed keywords so batch continues
  }
}
```

## 2. Abort/Race Conditions in Keyword Explorer

**Issue**: Long-running lookups (`runCompare`, `generate`) had no cancellation mechanism. User could start a lookup, switch stores, and have stale requests pollute state.

**Fix**: `apps/web/src/pages/aso/KeywordExplorerPage.tsx`
- Added `abortRef` to track active AbortController
- Each `runCompare`/`generate` creates new controller, cancels previous
- Cleanup on unmount calls `abort()`
- State updates check `!ctrl.signal.aborted` before applying results

**Code**:
```typescript
const abortRef = useRef<AbortController | null>(null);

// In runCompare/generate:
abortRef.current?.abort();
const ctrl = new AbortController();
abortRef.current = ctrl;

// Pass signal to async operations
const fetched = await compareKeywords(..., ctrl.signal);

// Check before state update
if (!mounted.current || ctrl.signal.aborted) return;
```

## 5. Data Freshness Labels (Cached vs Fresh)

**Issue**: UI showed "live store search" for all data, regardless of cache age. User couldn't tell if metrics were current or stale.

**Fixes**:

### A. Added `computedAt` timestamp
- `packages/db/src/queries/keywords.ts`: Include `computedAt` in returned KeywordDifficulty
- `packages/types/src/index.ts`: Added optional `computedAt?: string` to interface
- `apps/web/src/lib/api/keywords.ts`: Added to normalize function

### B. Freshness detection helpers
```typescript
export function isDataFresh(computedAt?: string): boolean {
  if (!computedAt) return false;
  const age = Date.now() - new Date(computedAt).getTime();
  const FRESH_THRESHOLD = 60 * 60 * 1000; // 1 hour
  return age < FRESH_THRESHOLD;
}

export function formatDataSource(computedAt?: string): string {
  if (!computedAt) return "store search";
  const fresh = isDataFresh(computedAt);
  return fresh ? "live store search" : "cached store search";
}
```

### C. UI Label Update
`apps/web/src/components/aso/KeywordBits.tsx`: Changed detail subtitle from hardcoded "live store search" to dynamic:
```typescript
<div className="kw-detail-sub">
  {kd.competingAppCount.toLocaleString()} competing apps · {formatDataSource(kd.computedAt)}
</div>
```
